# Portfolio Frontend

## 概要

個人ポートフォリオサイトのフロントエンドプロジェクトです。モダンなwebサイト構築のために、静的サイトジェネレーターであるAstroを採用しています。

## 技術スタック

### フレームワーク

- **Astro 5.16.6** - 静的サイトジェネレーター
- **TypeScript** - 型安全な開発

### スタイリング

- **Tailwind CSS 4.1.18** - ユーティリティファーストCSS フレームワーク

### 開発ツール

- **ESLint** - コード品質の保持
- **Prettier** - コードフォーマット
- **Husky** - Gitフック管理
- **Commitlint** - コミットメッセージの規約

## 技術選定理由：なぜAstroを選んだのか

### Next.jsと比較したAstroの強み

#### 1. **パフォーマンス優位性**

- **Zero JavaScript by default**:
  Astroは必要な部分でのみJavaScriptを読み込み、ページの初期読み込み速度が向上
- **島（Islands）アーキテクチャ**: インタラクティブな部分のみを選択的にhydrateできるため、不要なJSバンドルサイズを削減
- Next.jsはSSRやCSRでReactを全体的に読み込むため、軽量性でAstroが優位

#### 2. **静的サイト生成に特化**

- **Built-in静的生成**: ポートフォリオのような静的コンテンツには最適化されている
- **ファイルベースルーティング**: シンプルで直感的なページ構造
- Next.jsも静的生成はできるが、Reactフレームワークとしての複雑さがある

#### 3. **フレームワーク非依存**

- **マルチフレームワーク対応**: React、Vue、Svelteなどを必要に応じて混在可能
- **段階的な導入**: 静的コンテンツから始めて、必要に応じてインタラクティブ要素を追加
- Next.jsはReactエコシステムに固定される

#### 4. **学習コストと開発体験**

- **シンプルな概念**: 静的サイト構築に必要な機能に集約
- **直感的なファイル構造**: `.astro`ファイルでコンポーネントを直感的に記述
- **優れたDX**: 開発者体験がシンプルで迷いが少ない

#### 5. **SEO・Webパフォーマンス**

- **Built-in SEO最適化**: メタタグ、サイトマップの自動生成
- **Lighthouse高スコア**: 軽量性により、パフォーマンス指標で優秀な結果
- **CDNフレンドリー**: 完全な静的ファイル出力でCDN配信が最適

## プロジェクト構成

```
src/
├── assets/           # 画像、フォントなどの静的アセット
├── features/         # 機能別コンポーネント
├── layouts/          # レイアウトテンプレート
├── pages/            # ルーティング対象のページ
├── shared/           # 共有コンポーネントやユーティリティ
└── styles/           # グローバルスタイル
```

## セットアップ

### 前提条件

- Node.js 18以上
- pnpm

### インストール

```bash
pnpm install
```

### 開発サーバー起動

```bash
pnpm dev
```

開発サーバーは `http://localhost:4321` で起動します。

### ビルド

```bash
pnpm build
```

### プレビュー

```bash
pnpm preview
```

## 開発フロー

### コード品質

- **ESLint**: `pnpm lint` でコード品質をチェック
- **Prettier**: `pnpm format` でコードフォーマット
- **自動修正**: `pnpm lint:fix` で自動修正可能な問題を解決

### Git フック

- **pre-commit**: コミット前にlintとformat checkを実行
- **commit-msg**: Conventional Commitsに従ったコミットメッセージの検証
